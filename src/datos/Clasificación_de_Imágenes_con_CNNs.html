📐 Tamaños de las Imágenes y Relación con la CNN

Cada imagen de CIFAR-10 tiene tamaño 32x32x3, lo que representa:

* 32 píxeles de alto
* 32 píxeles de ancho
* 3 canales de color (RGB)

Este tamaño encaja perfectamente con la entrada de una Conv2D. Después de dos capas de pooling, el tamaño se reduce a 8x8, y al aplicar 64 filtros, la entrada al Flatten será de 8x8x64 = 4096 neuronas.
 
🧠 Justificación Detallada de la Arquitectura CNN

A continuación detallamos por qué elegimos esta arquitectura específica para trabajar con el dataset CIFAR-10.

🔹 Entrada: input_shape=(32, 32, 3)
Cada imagen tiene resolución 32x32 píxeles y 3 canales (RGB), lo cual es ideal para redes pequeñas y rápidas.
🔹 Capa 1: Conv2D(32, (3, 3))
Se aplican 32 filtros pequeños de 3x3, los cuales son suficientes para detectar bordes, contornos y texturas simples.
Activa regiones locales sin perder la resolución original.
🔹 Capa 2: MaxPooling2D(2, 2)
Reduce la dimensión espacial de la imagen a la mitad: 32x32 → 16x16.
Esto reduce el número de parámetros, previene sobreajuste y acelera el entrenamiento.
🔹 Capa 3: Conv2D(64, (3, 3))
Aplica más filtros (64), permitiendo aprender características más complejas como formas, esquinas, o patrones específicos.
🔹 Capa 4: MaxPooling2D(2, 2)
Vuelve a reducir las dimensiones: 16x16 → 8x8.
🔹 Capa 5: Flatten()
Convierte el tensor 8x8x64 en un vector de tamaño 4096 que puede ser interpretado por capas densas tradicionales.
🔹 Capa 6: Dense(64)
Una capa intermedia para sintetizar la información aprendida en la convolución.
Su tamaño es una decisión empírica: suficientemente grande para aprender, suficientemente pequeña para evitar sobreajuste.
🔹 Capa 7: Dense(10, activation='softmax')
Salida con 10 neuronas, una por clase en CIFAR-10.
softmax transforma los valores en probabilidades para clasificar la imagen.

🧠 Reflexión: ¿Por qué esta arquitectura?

* Es eficiente: pocas capas, rápido de entrenar.
* Evita sobreajuste inicial: no tiene parámetros excesivos.
* Sirve como base para entender y experimentar con arquitecturas más complejas como VGG o ResNet.
 
🧱 Justificación de la Arquitectura

Nuestra red tiene la siguiente estructura:

* Conv2D(32) para captar bordes simples
* MaxPooling2D(2,2) para reducir resolución
* Conv2D(64) para aprender patrones más complejos
* MaxPooling2D(2,2)
* Flatten() para transformar a vector
* Dense(64) para clasificación intermedia
* Dense(10) con softmax para salida final

Es una red eficiente para imágenes pequeñas, como CIFAR-10.

🔐 Modelo CNN con Dropout y L2
🔹 Capa 1: Conv2D(32, (3,3), activation='relu', kernel_regularizer=l2(0.001))
* 32 filtros de tamaño 3x3.
* Activa regiones locales (como bordes o esquinas).
* La regularización L2 (λ = 0.001) penaliza pesos grandes, ayudando a evitar sobreajuste.
* input_shape=(32,32,3) se refiere al tamaño de las imágenes en CIFAR-10.
🔹 Capa 2: MaxPooling2D(2,2)
* Reduce el tamaño espacial: 32x32 → 16x16.
* Retiene características más importantes (usualmente el valor máximo).
🔹 Capa 3: Dropout(0.25)
* Apaga aleatoriamente el 25% de las neuronas durante el entrenamiento.
* Obliga a la red a no depender demasiado de neuronas específicas, mejorando la generalización.
🔹 Capa 4: Conv2D(64, (3,3), activation='relu', kernel_regularizer=l2(0.001))
* Segundo bloque convolucional, ahora con 64 filtros.
* Captura patrones más complejos (formas, texturas combinadas).
🔹 Capa 5: MaxPooling2D(2,2) → reducción de 16x16 → 8x8
🔹 Capa 6: Dropout(0.25)
* Repite la misma lógica: evitar coadaptación entre neuronas.
🔹 Capa 7: Flatten()
* Convierte el tensor 8x8x64 en un vector plano de tamaño 4096.
* Prepara la entrada para las capas densas (fully connected).
🔹 Capa 8: Dense(64, activation='relu', kernel_regularizer=l2(0.001))
* Capa completamente conectada con 64 neuronas.
* Aquí se combinan todas las características aprendidas.
* Se aplica L2 regularization también.
🔹 Capa 9: Dropout(0.5)
* Apaga el 50% de las neuronas para mayor regularización.
* Esto es común en la primera capa Dense antes de la salida.
🔹 Capa 10: Dense(10, activation='softmax')
* Capa de salida con 10 neuronas, una para cada clase de CIFAR-10.
* softmax produce una distribución de probabilidad sobre las clases.

✅ ¿Por qué usar Dropout y L2 juntos?

| Técnica | Qué hace | Cuándo ayuda |
| :------ | :----------------------- | :------------------------------------ |
| L2 | Penaliza pesos grandes | Cuando hay muchas neuronas/pesos |
| Dropout | Apaga conexiones aleatorias | Cuando hay coadaptación (sobreajuste) |

Ambas técnicas actúan en diferentes niveles, por lo que se complementan muy bien.

Esta red tiene una arquitectura más profunda, incorpora normalización por lotes y mayor capacidad de aprendizaje, lo que permite mejorar el rendimiento sobre el dataset CIFAR-10. Está compuesta por tres bloques convolucionales, cada uno seguido de normalización, pooling y dropout.

🔷 Estructura General del Modelo

Bloque 1:

* 2 capas convolucionales con 32 filtros y activación ReLU.
* Normalización después de cada convolución.
* MaxPooling para reducir tamaño espacial.
* Dropout del 25%.

Bloque 2:

* 2 capas convolucionales con 64 filtros, también con normalización.
* MaxPooling y Dropout (25%).

Bloque 3:

* 1 capa convolucional con 128 filtros.
* Normalización y MaxPooling.
* Dropout más fuerte (30%).

Clasificador final:

* Flatten para convertir la salida a un vector.
* Una capa densa (fully connected) con 128 neuronas y activación ReLU.
* Dropout del 50% para alta regularización.
* Capa de salida con 10 neuronas (una por clase), usando softmax.

🎯 Objetivo de este diseño

* Aumentar la capacidad del modelo para aprender características jerárquicas de las imágenes.
* Reducir la sensibilidad al sobreajuste mediante Dropout y L2.
* Estabilizar y acelerar el entrenamiento con Batch Normalization.

from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, MaxPooling2D, Dropout, Flatten, Dense, BatchNormalization
from tensorflow.keras import regularizers

model_mejorado = Sequential([
    # Bloque 1
    Conv2D(32, (3,3), activation='relu', padding='same', input_shape=(32,32,3)),
    BatchNormalization(),
    Conv2D(32, (3,3), activation='relu', padding='same'),
    BatchNormalization(),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.25),

    # Bloque 2
    Conv2D(64, (3,3), activation='relu', padding='same'),
    BatchNormalization(),
    Conv2D(64, (3,3), activation='relu', padding='same'),
    BatchNormalization(),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.25),

    # Bloque 3
    Conv2D(128, (3,3), activation='relu', padding='same'),
    BatchNormalization(),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.3),

    # Clasificador
    Flatten(),
    Dense(128, activation='relu', kernel_regularizer=regularizers.l2(0.001)),
    Dropout(0.5),
    Dense(10, activation='softmax')
])
