2. CreaciГіn de la Red Neuronal Simple:



Estructura: una capa oculta con 10 neuronas y una capa de salida con 3 neuronas (para 3 clases en el dataset Iris).
 

Definir el modelo

In [4]:
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense

model = Sequential([
    Dense(10, input_shape=(4,), activation='relu'),  # Capa oculta con 10 neuronas
    Dense(3, activation='softmax')  # Capa de salida con 3 neuronas (para 3 clases)
])

# CompilaciГіn del modelo
model.compile(optimizer='adam', loss='categorical_crossentropy', metrics=['accuracy'])

ВҝPor quГ© usamos ReLU y Softmax como funciones de activaciГіn?



* ReLU (Rectified Linear Unit): Se utiliza comГәnmente en las capas ocultas de las redes neuronales por varias razones:
* Simplicidad y Eficiencia Computacional: Es muy fГЎcil de calcular ($f(x) = \max(0, x)$), lo que acelera el entrenamiento.
* AtenuaciГіn del Problema del Gradiente Desvaneciente: A diferencia de las funciones sigmoide o tanh, ReLU no satura para valores positivos grandes, lo que ayuda a mitigar el problema del gradiente desvaneciente en redes profundas.
* Esparcidad: Introduce esparcidad en la red al hacer que algunas neuronas tengan una activaciГіn de cero, lo que puede ayudar a que el modelo sea mГЎs eficiente y aprenda caracterГӯsticas mГЎs robustas.

* Softmax: Se utiliza exclusivamente en la capa de salida de los modelos de clasificaciГіn multiclase (cuando hay mГЎs de dos clases).
* Probabilidades: Transforma las salidas de la capa anterior en una distribuciГіn de probabilidad sobre las diferentes clases. La suma de las salidas de Softmax para todas las clases es 1, lo que permite interpretarlas como la probabilidad de que una entrada pertenezca a cada clase.
* ClasificaciГіn Multiclase: Es ideal para tareas donde una entrada debe ser asignada a una y solo una de mГәltiples categorГӯas.
 

ВҝCГіmo influye el nГәmero de Г©pocas y el batch size en el desempeГұo del modelo?



NГәmero de Гүpocas (Epochs)



El nГәmero de Г©pocas se refiere a la cantidad de veces que el algoritmo de entrenamiento verГЎ todo el conjunto de datos de entrenamiento. Una Г©poca completa significa que cada ejemplo en el conjunto de entrenamiento ha tenido la oportunidad de actualizar los pesos del modelo.

Influencia en el DesempeГұo:

* Demasiadas Гүpocas (Overfitting - Sobreajuste):
* Si el nГәmero de Г©pocas es demasiado alto, el modelo puede empezar a "memorizar" el ruido y las caracterГӯsticas especГӯficas del conjunto de entrenamiento. Esto lleva a un rendimiento muy bueno en el conjunto de entrenamiento, pero un rendimiento pobre en datos nuevos (conjunto de prueba), lo que se conoce como sobreajuste.
* La pГ©rdida en el conjunto de entrenamiento seguirГЎ disminuyendo, pero la pГ©rdida en el conjunto de prueba (validaciГіn) comenzarГЎ a aumentar despuГ©s de cierto punto.

* Pocas Гүpocas (Underfitting - Subajuste):
* Si el nГәmero de Г©pocas es demasiado bajo, el modelo no tendrГЎ suficiente tiempo para aprender los patrones subyacentes en los datos. No convergerГЎ a una soluciГіn Гіptima y tanto la pГ©rdida de entrenamiento como la de validaciГіn serГЎn altas, lo que indica un subajuste.
* El modelo no alcanzarГЎ su mГЎximo potencial de rendimiento.

* Equilibrio: El objetivo es encontrar un nГәmero de Г©pocas donde el modelo haya aprendido lo suficiente para generalizar bien a nuevos datos sin sobreajustarse. Esto a menudo se determina monitoreando la pГ©rdida o la precisiГіn en un conjunto de validaciГіn y deteniendo el entrenamiento cuando el rendimiento en este conjunto deja de mejorar (tГ©cnica de "early stopping").

Batch Size (TamaГұo del Lote)



El batch size es el nГәmero de ejemplos de entrenamiento que se procesan antes de que los parГЎmetros internos del modelo se actualicen. En otras palabras, es la cantidad de muestras que se alimentan a la red en una sola pasada de propagaciГіn hacia adelante y hacia atrГЎs.

Influencia en el DesempeГұo:

* Batch Size Grande:
* Procesamiento mГЎs RГЎpido por Гүpoca: Se requieren menos actualizaciones de los pesos por Г©poca, lo que puede acelerar el tiempo total de una Г©poca.
* EstimaciГіn de Gradiente mГЎs Estable: El gradiente calculado a partir de un lote grande es una aproximaciГіn mГЎs precisa del gradiente real del conjunto de datos completo, lo que puede llevar a una trayectoria de optimizaciГіn mГЎs suave y estable.
* Riesgo de MГӯnimos Locales: Puede dificultar que el modelo escape de mГӯnimos locales (puntos donde la pГ©rdida es baja pero no la mГЎs baja posible) en la superficie de pГ©rdida.
* Requiere mГЎs Memoria: Un lote mГЎs grande necesita mГЎs memoria, lo que puede ser una limitaciГіn.

* Batch Size PequeГұo (Stochastic Gradient Descent o mini-batches):
* Procesamiento mГЎs Lento por Гүpoca: Se realizan mГЎs actualizaciones de los pesos por Г©poca, lo que puede ralentizar el tiempo total de una Г©poca, pero a menudo conduce a una convergencia mГЎs rГЎpida hacia una buena soluciГіn.
* EstimaciГіn de Gradiente mГЎs Ruidosa: El gradiente es mГЎs ruidoso y menos preciso, lo que puede introducir mГЎs fluctuaciones en la trayectoria de optimizaciГіn. Estas fluctuaciones, sin embargo, pueden ayudar al modelo a escapar de mГӯnimos locales y encontrar mejores soluciones.
* RegularizaciГіn ImplГӯcita: La naturaleza ruidosa del gradiente puede actuar como una forma de regularizaciГіn, ayudando a prevenir el sobreajuste.
* Menos Memoria: Requiere menos memoria, lo que permite entrenar modelos mГЎs grandes en hardware con recursos limitados.

* Equilibrio: La elecciГіn del batch size es un hiperparГЎmetro crucial. Un batch size muy pequeГұo puede hacer que el entrenamiento sea muy lento y ruidoso, mientras que un batch size muy grande puede hacer que el modelo se estanque en un mГӯnimo local o no generalice bien. A menudo, se encuentra un equilibrio en tamaГұos de lote intermedios (mini-batches), como 32, 64 o 128, que ofrecen un buen compromiso entre la eficiencia computacional y la calidad del aprendizaje.
